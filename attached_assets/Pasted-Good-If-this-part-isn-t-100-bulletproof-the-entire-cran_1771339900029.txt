# 🏗 STEP 1 — Lock Time Before Calculation

When Tom presses:

> Complete Job

The system should NOT immediately generate invoice.

It should:

### 1️⃣ Freeze Time Entries

```
WorkerTime {
  worker_id
  job_id
  start_time
  end_time
  source: auto_geofence | manual
  edited: true/false
  edited_by
  edit_reason
}
```

### 2️⃣ Validate

System checks:

* Is anyone still clocked in?
* Any overlapping time?
* Any negative duration?
* Any missing end time?

If issue:

> “Time conflict detected. Review before completion.”

No invoice generation until clean.

# 🏗 STEP 2 — Rate Snapshot System (CRITICAL)

This is where most apps fail.

Rates must be SNAPSHOTTED at job time.

If:

* Operator rate changes next week
* Crane rate updated next month

Old jobs must not change.

So when job is created or started, you store:

```
JobRateSnapshot {
  crane_rate_at_time
  operator_rate_at_time
  dogman_rate_at_time
  travel_rate_at_time
}
```

Never calculate from “live” rates.

Always from snapshot.

---

# 🏗 STEP 3 — Calculation Engine (Deterministic)

Calculation must use:

```
duration_minutes = end_time - start_time
billable_hours = round(duration_minutes / 60, billing_rule)
```

Billing rule options:

* Round up to nearest 15 min
* Round to 30 min
* Exact minute
* Minimum 4 hour call-out

Crane businesses often have minimum callouts.

So system must support:

```
if billable_hours < minimum_hours:
    billable_hours = minimum_hours
```
