The 3 upgrades that will make it feel instantly more premium

1) Make every screen have a “Hero + Context”

Right now some pages are “Title + empty card”.

Upgrade rule:
	•	Every screen starts with either:
	•	a hero metric (count / money / status), or
	•	a hero action (“Create Invoice”, “Add Document”), or
	•	a hero summary (“3 issues need attention”)

2) Add “Contextual actions” everywhere

Premium mobile apps feel advanced because you can do stuff without digging.

Examples:
	•	Long press job card → quick actions (Start, Invoice, Message, Navigate)
	•	Swipe invoice → Send reminder / Mark paid
	•	Tap & hold team member → Call / Message / View jobs

3) Add “native feel”
	•	Haptics on primary actions
	•	Smooth transitions
	•	Loading skeletons instead of blank spinners
	•	Pull-to-refresh
	•	Sticky section headers
	•	Bottom sheets for secondary actions (instead of new screens)

Copy-paste prompt to Replit (use this)

Use this EXACTLY:

Prompt:

“Upgrade the existing JobRunner mobile app (do NOT redesign navigation or change the overall layout).
Goal: make the app feel premium, more native, and closer to the web app’s functionality.

Do a screen-by-screen audit and refactor.

Deliverables
	1.	A checklist of issues per screen:

	•	Visual hierarchy problems (no hero, too flat, too empty)
	•	Missing ‘advanced’ functionality vs web
	•	Missing contextual actions
	•	Missing native interaction patterns

	2.	Implement improvements across ALL screens:

Visual polish rules
	•	Introduce a hero section at top of every screen (metric / summary / primary action)
	•	Use background layering to separate sections (subtle tinted background blocks)
	•	Add subtle elevation (soft shadow) and consistent corner radius
	•	Enforce consistent spacing rhythm (24px between major sections, 16px inside cards)
	•	Improve empty states with illustration + clear next action + short guidance

Native feel rules
	•	Add pull-to-refresh where lists exist
	•	Add skeleton loaders for loading states
	•	Add haptic feedback on primary actions
	•	Replace secondary actions with bottom sheets (ActionSheet) instead of pushing new screens
	•	Add swipe actions on list items where appropriate (e.g., invoice reminders, mark complete)

Feature depth rules (match web where possible)
	•	Add advanced filtering + sorting on key lists
	•	Add bulk actions on lists (multi-select)
	•	Add quick actions on cards (long press / kebab menu)
	•	Add deep linking between modules (job → invoice → payment → client)
	•	Add inline editing for small fields (status, notes, tags)

Constraints
	•	Keep the current navigation tabs and overall structure
	•	Keep current brand colors and typography direction
	•	Focus on premium feel + advanced workflow depth
	•	Make changes consistent across every screen, not just one

Start by producing the audit checklist, then implement changes iteratively.”
