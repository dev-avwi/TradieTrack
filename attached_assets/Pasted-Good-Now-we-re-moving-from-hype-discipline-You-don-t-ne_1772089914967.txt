🔴 LEVEL 1 — CORE BUSINESS LOGIC (Non-Negotiable)

Job Lifecycle
	•	Create job
	•	Edit job
	•	Delete job (soft delete, not hard delete)
	•	Convert quote → job
	•	Convert job → invoice
	•	Cancel job
	•	Duplicate job
	•	Job status tracking (draft / scheduled / in progress / completed / invoiced)

Client Management
	•	Create client
	•	Edit client
	•	Duplicate client prevention
	•	Merge duplicate clients
	•	Client job history accurate

Data Integrity
	•	No duplicate invoice numbers
	•	Unique ID generation collision-safe
	•	Timestamps correct timezone handling
	•	All records tied to correct business (multi-tenant isolation)

⸻

🔴 LEVEL 2 — FINANCIAL ACCURACY (Trust Layer)

GST / Tax
	•	GST calculated correctly
	•	Rounding consistent
	•	Discounts handled correctly
	•	Line-item tax accurate

Invoices
	•	Partial payments supported
	•	Payment status tracking
	•	Invoice edits re-calc totals correctly
	•	Prevent double-send
	•	Invoice numbering sequence consistent
	•	PDF output matches on-screen totals

Edge Cases
	•	Editing quote after job created
	•	Editing job after invoice draft created
	•	Removing line item recalculates correctly
	•	Deleting client doesn’t delete invoices

If money math fails once → trust destroyed.

⸻

🔴 LEVEL 3 — MULTI-USER / TEAM LOGIC
	•	Role-based permissions (owner, admin, staff)
	•	Staff can’t see other businesses
	•	Prevent simultaneous overwrite (concurrency control)
	•	Activity log per job
	•	Change history recorded (Proof Pack integrity)
	•	Location tracking only for active shifts

⸻

📱 MOBILE READINESS (Real World Conditions)

Connectivity
	•	Graceful handling of weak signal
	•	Retry logic on failed API calls
	•	Prevent data loss if app crashes
	•	No infinite loading states

Device Handling
	•	Background resume safe
	•	GPS battery drain optimized
	•	Camera upload compression
	•	Large image handling stable
	•	Push notifications reliable

UX Safety
	•	Confirmation before destructive actions
	•	Clear error messages
	•	No hidden critical actions
	•	Tap protection against double submits

⸻

🌐 WEB APP READINESS
	•	Responsive layouts
	•	Works on Chrome, Safari, Edge
	•	No console errors
	•	No layout shifts breaking forms
	•	Keyboard navigation usable
	•	Loading states present
	•	Forms validate correctly

⸻

🔧 INFRASTRUCTURE & BACKEND
	•	Database indexing optimized
	•	No N+1 query problems
	•	Rate limiting on APIs
	•	API authentication verified
	•	Backup system confirmed
	•	Restore test performed
	•	Environment variables secure
	•	No secrets in repo

⸻

🔌 INTEGRATIONS HARDENING

Twilio
	•	SMS idempotency (no duplicate sends)
	•	Inbound messages attached to correct job
	•	Failure handling
	•	SMS cost caps
	•	Logging of all send attempts

AI Quotes
	•	AI output labeled as draft
	•	User must confirm before saving
	•	Rate limiting
	•	No direct write to financial totals without review
	•	Error fallback if AI fails

Maps / Dispatch
	•	Location update throttling
	•	Map fallback if API fails
	•	Accurate timezone handling
	•	Prevent ghost jobs on board

⸻

🔐 SECURITY
	•	JWT/session validation
	•	Secure storage on mobile
	•	HTTPS enforced
	•	Input sanitization
	•	SQL/NoSQL injection prevention
	•	File upload validation

	•	Audit log for admin actions
