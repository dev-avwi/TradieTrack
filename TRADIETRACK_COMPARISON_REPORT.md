## Critical Gaps Analysis

### HIGH PRIORITY (Required Before Launch)

1. **Stripe AU Platform Configuration**
   - **Issue:** Australia (AU) not enabled in Stripe Express settings
   - **Impact:** Blocks all payment functionality for Australian tradies
   - **Fix:** One-time configuration at https://dashboard.stripe.com/settings/applications/express
   - **Effort:** 5 minutes (platform admin task)

2. **Accounting Software Integration**
   - **Issue:** No MYOB, Xero, or QuickBooks integration
   - **Impact:** Tradies need to manually transfer financial data
   - **Fix:** Implement Xero/MYOB API integration
   - **Effort:** 2-4 weeks development

### MEDIUM PRIORITY (Post-Launch Improvements)

3. **Photo Markup/Annotations**
   - **Issue:** Cannot draw on photos to highlight issues
   - **Impact:** Less visual communication capability
   - **Fix:** Add canvas-based drawing tool on photo viewer
   - **Effort:** 1-2 weeks

4. **Video Capture**
   - **Issue:** No video recording on job sites
   - **Impact:** Cannot document complex issues visually
   - **Fix:** Add video recording to job documentation
   - **Effort:** 1 week

5. **Voice AI Assistant (Siri)**
   - **Issue:** No voice-activated assistant
   - **Impact:** Less hands-free capability
   - **Fix:** Implement Siri Shortcuts for iOS
   - **Effort:** 2-3 weeks

6. **Route Optimization**
   - **Issue:** No multi-stop route planning
   - **Impact:** Manual route planning for tradies with multiple jobs
   - **Fix:** Integrate Google Maps/HERE routing API
   - **Effort:** 1-2 weeks

### LOW PRIORITY (Nice to Have)

7. **Bulk Job Operations**
   - **Issue:** Cannot select and modify multiple jobs at once
   - **Fix:** Add multi-select and bulk action toolbar
   - **Effort:** 1 week

8. **Room Scan (iOS LiDAR)**
   - **Issue:** No spatial measurement tool
   - **Fix:** Integrate ARKit for iOS
   - **Effort:** 2-3 weeks

```

   |                         |                           |
   |-- Create Job ---------->|                           |
   |                         |-- Job in DB ------------->|
   |                         |<--- Delta Sync Request ---|
   |                         |--- New Job Data --------->|

```

**Verified:** All CRUD operations sync correctly between web and mobile within seconds when online, and queue for sync when offline.

## Photo Upload Verification

### Upload Flow

1. **Mobile:** User takes photo with camera
2. **Mobile:** FormData created with photo file
3. **Mobile:** POST to `/api/jobs/{id}/photos`
4. **Server:** Receives multipart/form-data
5. **Server:** Uploads to Google Cloud Storage (GCS)
6. **Server:** Generates signed URL for access
7. **Server:** Stores photo record in database
8. **Mobile:** Receives photo URL, updates UI

**Cloud Storage:** All photos uploaded to GCS bucket under `private/jobs/{jobId}/` path with signed URLs for secure access.

**Offline:** Photos captured offline are cached locally and uploaded when connectivity restores via sync queue.
